Polyfill as-a-service detects the current browser using the User-Agent header and bundles all the necessary polyfills into a single JS file, effectively polyfilling the browser itself. It also keeps track of the best polyfills and the relevant browser versions. You may now use the latest JS features and have it work on essentially all browsers with minimal effort.

Add the following code your HTML page:

<script src="https://cdn.jsdelivr.net/polyfills/polyfill.js"></script>

And you're good to go! Use Promise, requestAnimationFrame(), and so on in any browser!

This polyfill service is hosted by jsDelivr at https://cdn.jsdelivr.net/polyfills/polyfill.js and maintained in the Polyfills GitHub Organization.

API

GET https://cdn.jsdelivr.net/polyfills/polyfill.js Â± [polyfills...]

Returns a JavaScript bundle of polyfills based on the current User-Agent. You may either include or exclude a ,-delimited list of polyfills. To include polyfills, use a +, otherwise, use a -. You select each polyfill by its slug listed below.

Examples:

<!-- RECOMMENDED: include everything but Element.prototype.classList and Element.prototype.matches(), both of which are handled by the dom4 polyfill! -->
<script src="https://cdn.jsdelivr.net/polyfills/polyfill.js-clsls,mtchs"></script>

<!-- only include dom-elements and promises -->
<script src="https://cdn.jsdelivr.net/polyfills/polyfill.js+domels,promise"></script>
<!-- include everything but promises and dom-elements -->
<script src="https://cdn.jsdelivr.net/polyfills/polyfill.js-domels,promise"></script>

Middleware

The long-term goal of this service is to embed it within your node.js application. Requiring the usage of a CDN adds DNS lookup and HTTP request latency. Ideally, you would SPDY or HTTP/2 push each polyfill bundle to the client within the same server, removing any latency. Polyfilling middleware for node.js frameworks are available with SPDY push support:

  • koa-polyfills for koa
  • polyfills-middleware for Express and any other (req, res, next) node.js framework

Polyfills

Polyfills are tracked in polyfills-db, particularly lib/polyfills.js. Feel free to make pull requests to add, change, or update the metadata of each polyfill. Only high-quality, well-tested polyfills are used. Correctness and robustness is preferred over the final bundle size.

To test whether all the relevant polyfills are being loaded correctly, open https://cdn.jsdelivr.net/polyfills/test.html.
